Key Patterns to Know

useSuspenseQuery — the project uses the Suspense pattern. There are no isLoading flags. Wrap consumers in <Suspense fallback={<Skeleton />}>. 

// Pattern from get-posts.js — mirror this exactly
import { queryOptions, useSuspenseQuery } from '@tanstack/react-query'
import { apiClient } from '@/lib/api-client'

export const getPostsQueryOptions = ({ page = 1 } = {}) =>
  queryOptions({
    queryKey: ['posts', { page }],
    queryFn: () => apiClient.get(`/posts?page=${page}&limit=10`),
  })

export function usePosts({ page = 1 } = {}) {
  return useSuspenseQuery(getPostsQueryOptions({ page }))
}

useMutation + cache invalidation — after a successful mutation, call queryClient.invalidateQueries so the UI refreshes automatically. 

import { useMutation, useQueryClient } from '@tanstack/react-query'

export function useCreateComment({ postId }) {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: (data) => apiClient.post(`/posts/${postId}/comments`, data),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['comments', postId] })
    },
  })
}

React 19 form actions — use useActionState with <form action={formAction}>, not onSubmit. 

import { useActionState } from 'react'

const [state, formAction] = useActionState(
  async (prevState, formData) => {
    try {
      await mutation.mutateAsync({ body: formData.get('body') })
      return { error: null }
    } catch (err) {
      return { error: err.message }
    }
  },
  { error: null }
)

return <form action={formAction}>...</form>

Context + Reducer — use createContext paired with useReducer for shared state. Expose the dispatch function in the context value. 

import { createContext, useReducer, useContext } from 'react'

const PostsContext = createContext(null)

function reducer(state, action) {
  switch (action.type) {
    case 'SET_SEARCH': return { ...state, search: action.payload }
    case 'SET_FILTER': return { ...state, filter: action.payload }
    default: return state
  }
}

export function PostsProvider({ children }) {
  const [state, dispatch] = useReducer(reducer, { search: '', filter: 'all' })
  return (
    <PostsContext.Provider value={{ ...state, dispatch }}>
      {children}
    </PostsContext.Provider>
  )
}

export function usePostsContext() {
  const ctx = useContext(PostsContext)
  if (!ctx) throw new Error('usePostsContext must be used inside PostsProvider')
  return ctx
}

API Reference  ·  api.oluwasetemi.dev

All requests use the base URL https://api.oluwasetemi.dev. The apiClient helper in src/lib/api-client.js handles the base URL and JSON parsing for you.

Method & Endpoint Description Response shape
GET /posts?page=1&limit=10 Paginated list of posts {'{ data: Post[], meta: { page, totalPages, total } }'}
GET /posts/:id Single post by id {'{ id, title, body, userId, createdAt }'}
GET /posts/:postId/comments Comments for a post {'{ data: Comment[], meta: { total } }'}
POST /posts/:postId/comments Create a comment. Body: {'{ body: string }'} {'{ id, body, postId, userId, createdAt }'} — 201

// Post object

  id:        string,          // e.g. "post-1"
